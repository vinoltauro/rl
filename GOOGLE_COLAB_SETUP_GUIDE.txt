Understanding the Notebook Structure
-------------------------------------

Section 1: Setup and Installation
   → Installs required packages (gymnasium, matplotlib)
   → Imports libraries

Section 2: Environment Setup  
   → Creates CartPole environment
   → Shows state and action spaces

Section 3: Discretization Configuration
   → Sets up bins for state discretization
   → Defines state bounds

Section 4: Q-Table Initialization
   → Creates the Q-table (32,000 Q-values)

Section 5: Hyperparameters
   → Learning rate, discount factor, epsilon, etc.

Section 6: Core Functions
   → choose_action(): Epsilon-greedy selection
   → update_q_table(): Q-learning update

Section 7: Training Loop ⭐ (THIS IS THE MAIN PART)
   → Trains for up to 15,000 episodes
   → Shows live progress updates
   → Updates plots every 500 episodes
   → Stops early when solved (around episode 2000-3000)

Section 8: Results Visualization
   → Comprehensive 4-plot figure
   → Training statistics
   → Performance metrics

Section 9: Test Trained Agent
   → Runs 10 test episodes
   → Shows final performance

Section 10-11: Optional Analysis & Visualization

Tips for Google Colab
----------------------

1. **Free GPU**: You don't need it for CartPole, but available if curious
   - Runtime → Change runtime type → GPU

2. **Session Timeout**: 
   - Colab disconnects after ~90 minutes of inactivity
   - Training takes ~5-10 minutes, so no problem here!

3. **Saving Your Work**:
   - File → Download → Download .ipynb
   - OR File → Save a copy in Drive

4. **Rerunning**:
   - Runtime → Restart runtime → Clears all variables
   - Then run cells again from top

5. **Keyboard Shortcuts**:
   - Shift+Enter: Run cell and move to next
   - Ctrl+Enter: Run cell and stay
   - Ctrl+M B: Add cell below
   - Ctrl+M A: Add cell above


Troubleshooting
---------------

Problem: "ModuleNotFoundError: No module named 'gymnasium'"
Solution: Run the first cell again (!pip install gymnasium)

Problem: Plots not showing
Solution: 
   - Make sure you ran all previous cells first
   - Check if matplotlib imported successfully

Problem: Training takes forever
Solution:
   - It should take 5-10 minutes
   - Check if early stopping is enabled (stop_when_solved = True)
   - If >20 minutes, something may be wrong - restart runtime

Problem: "Runtime disconnected"
Solution:
   - Click "Reconnect"
   - Run all cells again
   - Training is fast enough this won't be an issue

Problem: Colab says "RAM/Disk full"
Solution:
   - Unlikely with this code (uses <100MB)
   - Runtime → Restart runtime → Try again
